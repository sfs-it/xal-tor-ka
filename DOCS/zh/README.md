# ⛬ Xal-Tor-Ka · `beta0.3`

*本文为官方英文文档的译文；如有出入，以英文版本为准。*

> 🌍 **语言：** [English（官方）](../../README.md) · [所有语言](../README.md)

**在你所有在线服务前设立的一道受守护的入口。** Xal-Tor-Ka 是你基础设施的
“数字门卫”：它在你发布的每一个站点和应用前放置一道统一的加固之门，并逐一决定
谁可以进入。不再需要把密码分散到各个服务里：规则集中在一处。

对于**决策者**而言，其价值很简单：你可以发布一个内部应用而无需将它暴露给整个
世界；你可以选择某个服务是**对所有人开放**、**仅限已登录用户**（含两步验证），
还是**仅限明确授权的人**；并且一切都通过一个**受保护的 Web 面板**进行管理。
用户可以使用内部凭据登录，也可以使用企业的 **Google** 或 **Microsoft** 账户登录。
系统会保留访问日志、抵御暴力破解尝试，并对自身配置进行备份。

## 适用对象

- 你运行着多个 Web 服务（内网、业务应用、仪表板、内部工具），希望拥有**一个
  受控的统一入口**，而不是分别保护每一个。
- 你希望**集中决定**谁能看到什么，配合第二安全因素和/或企业登录
  （Google/Microsoft），而**无需修改各个服务**。
- 你希望暴露在互联网上的唯一东西是一道**盾牌**，而真正的服务隐藏在它之后。

## 你能得到什么

- **一道统一的加固之门**：基础设施的其余部分无法从互联网直接访问。
- 每个服务有**三种访问级别**：公开 · 仅限已登录用户（2FA） · 仅限白名单中的人。
- **使用 Google / Microsoft 登录**，或使用内部凭据登录。
- **一个受保护的 Web 面板**，用于管理服务、用户和权限。
- **运维安全**：访问日志（可接入入侵防御系统）、自动配置备份、服务健康监控。
